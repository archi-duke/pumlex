# PlantUML Layout Editor (plantumlEx)

## 1. 배경 (Background)

PlantUML은 텍스트 기반 DSL로 다이어그램을 빠르게 생성할 수 있다는 강력한 장점이 있지만, 자동 배치(auto-layout) 엔진(GraphViz/Smetana 등)에 전적으로 의존한다. 이로 인해 다음과 같은 한계가 존재한다.

- 사용자가 원하는 위치에 노드(클래스, 컴포넌트, 액터 등)를 배치하기 어렵다.
- `together`, `left of`, `-[hidden]-` 같은 우회적 트릭에 의존해야 하며, 복잡한 다이어그램일수록 레이아웃을 제어하기 어렵다.
- 발표 자료, 문서, 아키텍처 설명용으로 사용할 때 "의도한 배치"와 "자동 생성된 배치" 사이의 간극이 크다.
- 한 번 생성된 SVG를 직접 편집하면 원본 PlantUML 소스와의 일관성이 깨져 재생성 시 수정 내용이 사라진다.

결과적으로 많은 사용자가 최종 배치를 맞추기 위해 Draw.io, Excalidraw 등 GUI 도구로 이동하거나, 하이브리드 워크플로로 생산성을 희생하고 있다.

## 2. 목적 (Purpose)

본 프로젝트 **plantumlEx**는 PlantUML의 텍스트 기반 생산성을 유지하면서, 생성된 다이어그램의 **시각적 배치를 직관적으로 미세조정(fine-tune)** 할 수 있는 편집 도구를 제공하는 것을 목표로 한다.

핵심 아이디어는 다음과 같다.

1. PlantUML 구문을 파싱하여 각 논리 요소(노드, 연결선, 그룹 등)에 대한 **메타정보**를 생성한다.
2. 생성된 SVG의 DOM 요소(`id`, `class`, 구조)와 메타정보를 **양방향 매핑**한다.
3. 사용자는 논리 이름(예: `ClassA`, `UserActor`)으로 요소를 선택하고, 드래그로 위치를 조정한다.
4. 노드 이동 시 **연결선(edge)의 경로를 실시간으로 재계산**하여 다이어그램의 시각적 일관성을 유지한다.
5. 조정된 배치 정보는 별도의 **레이아웃 오버레이**로 저장되어, 원본 PlantUML 소스를 훼손하지 않고 재생성 가능하다.

## 3. 주요 기능 (Main Features)

### 3.1 PlantUML 파싱 및 메타정보 생성
- PlantUML 소스에서 노드, 연결선, 그룹(package/namespace 등) 등 논리 요소를 식별한다.
- 각 요소에 안정적인 논리 ID(logical id)를 부여한다.
- 요소 간 관계(소스-타겟, 포함관계)를 그래프 형태의 메타 모델로 구성한다.

### 3.2 SVG 요소 매핑
- PlantUML이 생성한 SVG를 파싱하여 DOM 요소를 추출한다.
- 각 논리 요소를 해당 SVG 요소(`<g>`, `<rect>`, `<path>` 등)의 `id`와 매핑한다.
- 매핑 결과는 `logicalId → svgId`, `svgId → logicalId`의 양방향 구조로 관리한다.

### 3.3 요소 선택 및 하이라이트
- 좌측 트리/리스트 패널에서 논리 이름으로 요소를 검색·선택할 수 있다.
- 캔버스에서 SVG 요소를 직접 클릭하여 선택할 수 있다.
- 선택된 요소는 외곽선, 색상 오버레이 등으로 시각적으로 강조된다.

### 3.4 드래그 기반 배치 조정
- 선택된 노드를 드래그하여 자유롭게 이동할 수 있다.
- 이동 중 실시간 프리뷰를 제공한다(고스트 또는 실제 요소의 즉시 이동).
- 필요 시 스냅(격자, 정렬 가이드) 기능을 제공한다.
- 다중 선택 및 그룹 이동을 지원한다.

### 3.5 연결선 재계산
- 노드가 이동하면 해당 노드에 연결된 모든 edge의 경로를 재계산한다.
- edge 종류(직선, 직각, 곡선)와 화살표 방향·스타일을 유지한다.
- 가능하면 다른 노드와의 겹침을 최소화하는 간단한 라우팅 로직을 적용한다.

### 3.6 레이아웃 저장 및 적용
- 조정된 위치 정보를 원본 PlantUML 소스와 분리된 **레이아웃 파일**(예: `*.layout.json`)로 저장한다.
- 동일 소스에 저장된 레이아웃을 적용하여 다이어그램을 재생성할 수 있다.
- 소스가 변경되어 요소가 추가·삭제된 경우, 기존 레이아웃에서 매칭 가능한 요소는 유지하고 나머지는 자동 배치로 폴백한다.

### 3.7 내보내기 (Export)
- 조정된 상태의 SVG/PNG를 내보낼 수 있다.
- 레이아웃 파일을 팀원과 공유하여 동일한 배치를 재현할 수 있다.

## 4. 범위 밖 (Out of Scope, 초기 버전 기준)

- PlantUML 소스 자체의 편집기(구문 편집은 외부 에디터 가정).
- 모든 PlantUML 다이어그램 유형 지원(초기에는 클래스/컴포넌트 다이어그램 중심).
- 자동 최적 배치 알고리즘(수동 조정이 1차 목표).
